# Qwen Code 專案概覽

## 專案簡介

**Qwen Code** 是一個開源的終端機 AI 代理工具，專為 [Qwen3-Coder](https://github.com/QwenLM/Qwen3-Coder) 優化。它能幫助開發者理解大型程式碼庫、自動化繁瑣工作，並加速軟體開發流程。

本專案基於 [Google Gemini CLI](https://github.com/google-gemini/gemini-cli) 開發，並針對 Qwen-Coder 模型進行了適配與優化。

---

## 核心功能

### 1. 多協定認證支援
- **Qwen OAuth（推薦）**：透過瀏覽器進行 OAuth 認證，每日提供 1,000 次免費請求
- **API-KEY**：支援 OpenAI、Anthropic、Google GenAI、阿里雲百煉等相容端點

### 2. 代理式工作流程
- 豐富的內建工具（Skills、SubAgents）實現完整的代理工作流
- 類似 Claude Code 的使用體驗
- 規劃模式（Plan Mode）支援結構化任務執行

### 3. 終端優先，IDE 友善
- 使用 Ink（React for CLI）構建的互動式終端 UI
- 支援 VS Code、Zed、JetBrains IDE 整合
- 支援無頭模式（Headless Mode）用於腳本和 CI/CD

### 4. 高度可擴展性
- MCP（Model Context Protocol）整合
- LSP（Language Server Protocol）支援
- 自訂技能（Skills）與子代理（Subagents）系統

## 建置與開發流程

### 系統需求
- **Node.js**：開發環境 ~20.19.0，生產環境 >=20.0.0
- **Git**
- **Docker 或 Podman**（選用，用於沙盒環境）

### 安裝與設定

```bash
# 複製並安裝
git clone https://github.com/QwenLM/qwen-code.git
cd qwen-code
npm install
```

### 建置指令

| 指令 | 說明 |
|------|------|
| `npm run build` | 建置所有套件 |
| `npm run build:all` | 建置所有套件 + 沙盒 + VSCode companion |
| `npm run build:packages` | 僅建置套件 |
| `npm run bundle` | 打包用於發布 |
| `npm run dev` | 開發模式（熱重載） |

### 執行

```bash
npm start           # 啟動互動式 CLI
npm run debug       # 除錯模式（--inspect-brk）
```

### 測試

```bash
npm run test                    # 執行單元測試
npm run test:e2e               # 執行整合測試
npm run test:integration:all   # 執行所有整合測試（不同沙盒模式）
npm run test:terminal-bench    # 終端基準測試
```

### 程式碼品質

```bash
npm run preflight   # 完整 CI 檢查（clean, install, format, lint, build, typecheck, test）
npm run lint        # 執行 ESLint
npm run format      # 執行 Prettier
npm run typecheck   # 類型檢查
```

## 核心架構模式

### 1. 雙套件架構

**CLI 套件（`@qwen-code/qwen-code`）：**
- 使用者面向的終端介面
- 輸入處理（文字、斜線指令、@檔案引用、!shell 指令）
- 歷史記錄管理與會話恢復
- 語法高亮的顯示渲染
- 主題與 UI 客製化

**Core 套件（`@qwen-code/qwen-code-core`）：**
- 後端邏輯與 API 通訊
- Prompt 建構與模型互動
- 工具註冊與執行
- 狀態管理
- 伺服器端設定

### 2. 工具系統

位於 `packages/core/src/tools/`，包含 50+ 工具實作：

| 工具類別 | 工具 |
|----------|------|
| 檔案操作 | `read-file`, `write-file`, `edit`, `glob`, `grep`, `ls` |
| Shell 執行 | `shell` |
| 網頁 | `web-fetch`, `web-search` |
| MCP | `mcp-client`, `mcp-client-manager`, `mcp-tool` |
| LSP | `lsp` |
| 使用者互動 | `askUserQuestion` |
| 任務管理 | `todoWrite`, `task`, `exitPlanMode` |
| 記憶 | `memoryTool` |
| 技能 | `skill` |

所有工具繼承 `BaseDeclarativeTool` 或實作工具介面。

### 3. 子代理系統

專門化 AI 代理的任務委派框架：
- 設定以 Markdown + YAML frontmatter 格式儲存
- 支援專案層級與使用者層級的子代理
- 事件驅動架構用於 UI 更新
- 位於 `packages/core/src/subagents/`

### 4. 技能系統

可重複使用的技能系統：
- 內建技能位於 `packages/core/src/skills/bundled/`
- 目前包含：`review` 技能
- 可透過設定擴展

### 5. 設定系統

分層式設定載入（依優先順序）：
1. 命令列參數
2. 環境變數
3. 專案設定（`.qwen/settings.json`）
4. 使用者設定（`~/.qwen/settings.json`）
5. 系統設定檔
6. 預設值

### 6. 模型註冊表

彈性的模型設定支援：
- 多種認證類型（OpenAI、Anthropic、Gemini、Qwen OAuth）
- 自訂 Base URL 和 API Key
- 生成設定（採樣參數、逾時）
- 模型能力（視覺支援）

## 開發規範

### 程式碼風格
- **嚴格 TypeScript**：啟用所有嚴格標記（`strictNullChecks`、`noImplicitAny` 等）
- **模組系統**：ES 模組（`"type": "module"`）
- **匯入風格**：Node.js 原生 ESM，使用 `.js` 副檔名
- **禁止跨套件相對匯入**：由 ESLint 強制執行

### 匯入模式

```typescript
// 套件內部 - 使用相對路徑
import { something } from './utils/something.js';

// 跨套件 - 使用套件名稱
import { Config } from '@qwen-code/qwen-code-core';
```

### 測試模式
- 單元測試與原始檔案共存（`.test.ts` 後綴）
- 整合測試位於獨立的 `integration-tests/` 目錄
- 使用 Vitest，啟用全域變數
- HTTP 使用 `msw` 模擬，檔案系統使用 `memfs`/`mock-fs`

### 提交規範
遵循 [Conventional Commits](https://www.conventionalcommits.org/)
範例：`feat(cli): Add --json flag to 'config get' command`
